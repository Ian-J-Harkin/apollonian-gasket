# Apollonian Gasket - Project Guide

This project is a hybrid application setup. The same codebase builds for **Desktop Browsers, Progressive Web Apps (PWAs), Android, and iOS** simultaneously.

## Repository Structure

The project has been restructured to use **Capacitor** (by Ionic), which wraps the web code into native mobile app shells.

*   `public/`: **This is where all your code lives.** You will continue to write and modify your HTML, CSS, JS, and PWA files here. Nothing changes for your day-to-day web development workflow!
*   `android/`: Native Android application folder (generated by Capacitor).
*   `ios/`: Native iOS application folder (generated by Capacitor).
*   `node_modules/`: Required dependencies for Capacitor (ignored by Git).
*   `capacitor.config.json`: Configuration telling the native apps to pull code from the `public/` directory.

---

## Developer Workflow

### 1. Editing Your Code
Just continue modifying the files inside the `public/` directory exactly like you were doing before. 

### 2. Updating the Native Apps (iOS / Android)
Whenever you make changes in the `public/` folder and want to update the native iOS/Android apps, open your terminal and run:

```powershell
npx cap sync
```

This command takes all the latest code from `public/` and injects it into the native apps.

### 3. Testing on a Real Device / Simulator
To test the native applications on an emulator or compile them for the App Stores, use the following commands:

*   **For Android:** Run `npx cap open android` to launch Android Studio.
*   **For iOS:** Run `npx cap open ios` to launch Xcode (Mac only).

From there, you just press "Play" to test the app on your phone.

## Deployment 

### Web Deployment (Vercel)
To push the app to the web, you just need to link your GitHub repository to Vercel:

1. Go to your **Vercel** dashboard.
2. Click **Add New Project**.
3. Select your GitHub repository (`apollonian-gasket`).
4. **CRITICAL:** Before clicking Deploy, click **Edit** on the "Root Directory" option and type in `public`. This tells Vercel where your web code lives.
5. Click **Deploy**.

### GitHub Operations
Your project is already initialized as a Git repository with a `.gitignore` to keep it clean. To push changes to GitHub in the future, simply run:

```powershell
git add .
git commit -m "Your update message"
git push
```
